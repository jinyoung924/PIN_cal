 Rolling PIN Estimation Project — Specification Document
================================================================================

■ 프로젝트 목적
  - PINstimation(R 패키지)을 이용하여 60일 롤링 윈도우 방식으로
    종목별 일별 PIN(Probability of Informed Trading)을 추정한다.
  - 틱 데이터에 이미 LR(Lee-Ready) 알고리즘으로 분류된 매수/매도 컬럼이 
    존재하므로, trade classification 단계를 건너뛰고 바로 집계 후 추정한다.

1-1. 경로 구조
  루트: E:\vpin_project_parquet\processing_data\
  하위: 국가코드로 시작하는 폴더들이 다수 존재

    E:\vpin_project_parquet\processing_data\
    ├── KOR_2019\
    │   ├── kor_2019_01.parquet
    │   ├── kor_2019_02.parquet
    │   └── ...

    │   └── ...
    └── USA_2021\
        └── ...

  - 폴더명: {국가코드}_{연도} 또는 유사 형태 (국가코드로 시작)
  - 파일명: {국가코드}_{연도}_{청크번호}.parquet
    (한 연도의 데이터를 n개로 분할한 파일)
  - 하나의 국가를 처리할 때, 해당 국가코드로 시작하는 모든 폴더 내의
    .parquet 파일을 전부 읽어야 한다.

1-2. 파일 형식
  - Apache Parquet

1-3. 컬럼 구조 (틱 데이터)
  ┌──────────┬──────────┬──────────────────────────────────────┐
  │ 컬럼명   │ 타입     │ 설명                                 │
  ├──────────┼──────────┼──────────────────────────────────────┤
  │ Date     │ date32   │ 거래 날짜 (YYYY-MM-DD)               │
  │ Time     │ time64   │ 거래 시각                            │
  │ Symbol   │ string   │ 종목 코드                            │
  │ Price    │ float64  │ 거래 가격                            │
  │ Volume   │ float64  │ 거래량                               │
  │ LR       │ int8     │ Lee-Ready 분류 결과                  │
  │          │          │   1 = 매수(buyer-initiated)          │
  │          │          │  -1 = 매도(seller-initiated)         │
  └──────────┴──────────┴──────────────────────────────────────┘

  ※ PIN 추정에 실제 필요한 컬럼: Date, Symbol, LR (3개만)
     Price, Volume, Time은 PIN 추정에는 사용하지 않는다.

2. 데이터 전처리 (집계)
================================================================================

  틱 → 일별 종목별 매수/매도 건수 집계

  집계 로직:
    - LR == 1  인 건수 → B (buyer-initiated trades)
    - LR == -1 인 건수 → S (seller-initiated trades)
    - GROUP BY: Symbol, Date
    - B + S == 0 인 날은 제거

  집계 결과 형태:
  ┌────────────┬────────┬─────┬─────┐
  │ Date       │ Symbol │ B   │ S   │
  ├────────────┼────────┼─────┼─────┤
  │ 2019-01-02 │ A005930│ 312 │ 287 │
  │ 2019-01-02 │ A000660│ 145 │ 162 │
  │ ...        │ ...    │ ... │ ... │
  └────────────┴────────┴─────┴─────┘

3. PIN 추정 방법
================================================================================

3-1. 모델
  - Original PIN model (Easley & O'Hara 1992, Easley et al. 1996)
  - R 패키지: PINstimation (CRAN)

3-2. 추정 함수
  - 기본: pin_ea()  — Ersan & Alici (2016) 초기 파라미터 알고리즘
  - 대안: pin_yz() (Yan & Zhang 2012), pin_gwj() (Gan et al. 2015),
          pin_bayes() (Griffin et al. 2021)

3-3. Likelihood Factorization
  - 기본: "LK" — Lin & Ke (2011), 계산 안정성 우수
  - 대안: "E" (Ersan 2016), "EHO" (Easley et al. 2010), "NONE"

3-4. 추가 파라미터
  - xtraclusters = 4 (초기 파라미터 세트 생성용 추가 클러스터)

3-5. 롤링 윈도우
  - 윈도우 크기: 60 거래일
  - 슬라이딩: 1일씩 이동
  - 기준일: 윈도우의 마지막 날짜
  - 즉, 날짜 d의 PIN = [d-59, d] 구간 60일 B/S 데이터로 추정
  - 관측치가 60일 미만인 종목은 건너뛴다.

3-6. PIN 수식 (참고)
  PIN = (α × μ) / (α × μ + ε_b + ε_s)
  
  파라미터:
    α (alpha) : 정보 이벤트 발생 확률
    δ (delta) : 정보 이벤트가 악재(bad news)일 확률
    μ (mu)    : 정보 거래 도착률
    ε_b       : 비정보 매수 도착률
    ε_s       : 비정보 매도 도착률

3-7. PIN Good/Bad 분해 (Brennan et al. 2016)
  pinG = α × (1 - δ) × μ / (α × μ + ε_b + ε_s)   (호재 정보거래)
  pinB = α × δ × μ / (α × μ + ε_b + ε_s)          (악재 정보거래)

3-8. 입력 형식 (pin_ea 함수)
  - data.frame 또는 2컬럼 데이터프레임: 첫째 열 = B, 둘째 열 = S
  - 행 수 = 윈도우 크기 (60)
  - 각 행 = 1 거래일

4. 출력 데이터
================================================================================

4-1. 경로 구조
  E:\vpin_project_parquet\R_output\{국가코드}\PIN\

    ├── PIN_KOR.parquet          ← 전체 결과 (전 종목 통합)

4-2. 메인 결과 컬럼 (PIN_{국가코드}.parquet)
  ┌───────────┬──────────┬──────────────────────────────────────┐
  │ 컬럼명    │ 타입     │ 설명                                 │
  ├───────────┼──────────┼──────────────────────────────────────┤
  │ date      │ Date     │ 추정 기준일 (윈도우 마지막 날짜)     │
  │ symbol    │ string   │ 종목 코드                            │
  │ win_start │ Date     │ 윈도우 시작일                        │
  │ PIN       │ float64  │ 추정된 PIN 값                        │
  │ pinG      │ float64  │ Good-news PIN 성분                   │
  │ pinB      │ float64  │ Bad-news PIN 성분                    │
  │ alpha     │ float64  │ 정보 이벤트 확률                     │
  │ delta     │ float64  │ 악재 확률                            │
  │ mu        │ float64  │ 정보 거래 도착률                     │
  │ eps_b     │ float64  │ 비정보 매수 도착률                   │
  │ eps_s     │ float64  │ 비정보 매도 도착률                   │
  │ loglik    │ float64  │ 로그우도 값                          │
  │ success   │ logical  │ 추정 성공 여부                       │
  └───────────┴──────────┴──────────────────────────────────────┘

4-3. 요약 CSV 컬럼 (PIN_summary_{국가코드}.csv)
  symbol, days, pin_mean, pin_med, pin_sd, pin_min, pin_max,
  pinG_mean, pinB_mean, alpha_mean, mu_mean

5. 처리 흐름 요약
================================================================================

  ┌─────────────────────────────────────────────────────┐
  │ 1. 국가코드로 시작하는 모든 폴더에서 parquet 수집   │
  │    (읽기 컬럼: Date, Symbol, LR 만)                 │
  └──────────────────────┬──────────────────────────────┘
                         ▼
  ┌─────────────────────────────────────────────────────┐
  │ 2. 틱 → 일별 집계                                   │
  │    GROUP BY (Symbol, Date)                          │
  │    B = count(LR == 1), S = count(LR == -1)         │
  └──────────────────────┬──────────────────────────────┘
                         ▼
  ┌─────────────────────────────────────────────────────┐
  │ 3. 종목별 롤링 60일 윈도우 PIN 추정                 │
  │    - pin_ea(df[i:(i+59), c("B","S")], "LK")        │
  │    - 윈도우 1일씩 슬라이딩                          │
  │    - 추정 실패 시 NA 기록, 다음 윈도우 계속         │
  └──────────────────────┬──────────────────────────────┘
                         ▼
  ┌─────────────────────────────────────────────────────┐
  │ 4. 결과 저장                                        │
  │    - 전체: PIN_{국가코드}.parquet                    │
  │    - 요약: PIN_summary_{국가코드}.csv                │
  └─────────────────────────────────────────────────────┘

6. 핵심 제약 / 주의사항
================================================================================

  - LR 컬럼을 그대로 사용 (trade classification 재수행 안 함)
  - 60일 미만 데이터 종목은 추정 불가 → 건너뜀
  - PIN 추정 실패 시 해당 윈도우는 NA 처리 후 계속 진행
  - parquet 읽기 시 필요 컬럼(Date, Symbol, LR)만 선택하여 메모리 절약
  - 폴더 탐색: BASE_IN 하위 1단계 폴더 중 국가코드로 시작하는 것 전부
